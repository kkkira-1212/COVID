# Git Authentication Fix Guide

## Problem
GitHub no longer supports password authentication. You need to use either:
1. Personal Access Token (PAT) with HTTPS
2. SSH keys with SSH URL

## Solution Options

### Option 1: Use Personal Access Token (PAT) - Recommended for HTTPS

#### Step 1: Create Personal Access Token on GitHub
1. Go to GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic)
2. Click "Generate new token (classic)"
3. Select scopes: at minimum `repo` (full control of private repositories)
4. Copy the token (you'll only see it once!)

#### Step 2: Update Git Remote URL
```bash
# Keep HTTPS but use token
git remote set-url origin https://<YOUR_TOKEN>@github.com/kkkira-1212/COVID.git
```

Or better, use credential helper:
```bash
# Configure credential helper to store token
git config --global credential.helper store

# Or use cache (temporary)
git config --global credential.helper 'cache --timeout=3600'

# When pushing, use username and token as password
git push
# Username: kkkira-1212
# Password: <paste your PAT here>
```

### Option 2: Use SSH (More Secure, Recommended)

#### Step 1: Generate SSH Key (if not exists)
```bash
# Check if SSH key exists
ls -la ~/.ssh/id_*.pub

# If not, generate new SSH key
ssh-keygen -t ed25519 -C "wangxin2004001@163.com"
# Press Enter for default location
# Optionally set a passphrase

# Display public key
cat ~/.ssh/id_ed25519.pub
```

#### Step 2: Add SSH Key to GitHub
1. Copy the public key output from above
2. Go to GitHub → Settings → SSH and GPG keys → New SSH key
3. Paste the key and save

#### Step 3: Change Remote URL to SSH
```bash
git remote set-url origin git@github.com:kkkira-1212/COVID.git
```

#### Step 4: Test SSH Connection
```bash
ssh -T git@github.com
# Should see: "Hi kkkira-1212! You've successfully authenticated..."
```

#### Step 5: Push
```bash
git push
```

### Option 3: Quick Fix - Use GitHub CLI
```bash
# Install gh CLI (if not installed)
# Then authenticate
gh auth login

# This will handle authentication automatically
git push
```

## Recommended: Use SSH (Option 2)
SSH is more secure and doesn't require entering credentials every time (with SSH agent).

## Quick Commands Summary

```bash
# Check current remote
git remote -v

# Switch to SSH (if you have SSH key set up)
git remote set-url origin git@github.com:kkkira-1212/COVID.git

# Or use HTTPS with token stored
git config --global credential.helper store
# Then push and enter token when prompted

# Verify remote URL
git remote -v
```
