# 数据对比分析报告

## 概述

对比了旧数据文件 (`sequences_day.pt`, `sequences_week.pt`) 和新处理的数据文件 (`data/processed/day_6feat.pt`, `data/processed/week_6feat.pt`)。

## 主要发现

### 1. Day数据 (sequences_day.pt vs day_6feat.pt)

#### ✅ 一致的部分：
- **样本数**: 20280 (一致)
- **窗口大小**: 14 (一致)
- **特征数**: 6 (一致)
- **特征列表**: 完全一致

#### ⚠️ 发现的问题：

1. **正样本数量大幅减少**
   - 旧数据: 551个正样本 (2.72%)
   - 新数据: 351个正样本 (1.73%)
   - **减少了200个正样本 (约36%的减少)**

2. **数据结构变化**
   - 旧数据: 使用 `sklearn StandardScaler` 进行标准化
   - 新数据: 使用 `feature_means` 和 `feature_stds` 手动标准化
   - 新数据添加了 `X_next` 字段
   - 新数据添加了 `fine_to_coarse_index` 映射

3. **数据分割比例略有不同**
   - 旧数据: train=13416, val=2366, test=4498
   - 新数据: train=14196, val=2028, test=4056
   - 虽然总数相同，但分割方式不同

### 2. Week数据 (sequences_week.pt vs week_6feat.pt)

#### ✅ 一致的部分：
- **特征数**: 6 (一致)
- **特征列表**: 完全一致

#### ⚠️ 发现的问题：

1. **样本数量不一致**
   - 旧数据: 2756个样本
   - 新数据: 2860个样本
   - **增加了104个样本**

2. **窗口大小不一致**
   - 旧数据: 窗口大小 = 8
   - 新数据: 窗口大小 = 4
   - **窗口大小减半**

3. **正样本数量大幅减少**
   - 旧数据: 510个正样本 (18.51%)
   - 新数据: 329个正样本 (11.50%)
   - **减少了181个正样本 (约35%的减少)**

4. **数据结构变化**
   - 旧数据: 使用 `sklearn StandardScaler` 进行标准化
   - 新数据: 使用 `feature_means` 和 `feature_stds` 手动标准化
   - 新数据添加了 `X_next` 字段

## 可能的原因分析

### 正样本减少的可能原因：

1. **标签生成逻辑可能发生变化**
   - 虽然 `create_outbreak_labels` 函数看起来相同，但可能：
     - 输入数据发生了变化（缺失值处理、数据清洗等）
     - 特征工程步骤影响了 `NewDeaths_return` 的计算
     - 数据聚合方式（`group_by_state`）可能有所不同

2. **数据预处理差异**
   - 旧数据可能使用了不同的数据源或预处理步骤
   - 缺失值处理方式可能不同

3. **时间范围或数据过滤**
   - 可能应用了不同的时间范围过滤
   - 可能应用了不同的数据质量过滤

### Week数据窗口大小变化：

- 旧数据使用窗口大小8（8周）
- 新数据使用窗口大小4（4周）
- 这可能是有意的设计变更，但需要确认是否符合预期

## 建议

1. **检查标签生成过程**
   - 对比旧数据和新数据在标签生成前的中间结果
   - 检查 `NewDeaths_return` 和 `NewDeaths` 的计算是否一致
   - 检查数据聚合后的结果是否一致

2. **检查数据预处理**
   - 对比特征工程步骤
   - 检查缺失值处理
   - 检查数据过滤条件

3. **验证窗口大小变更**
   - 确认Week数据的窗口大小从8改为4是否是有意的
   - 如果是有意的，需要更新相关文档和模型配置

4. **数据一致性检查**
   - 建议创建一个脚本来验证数据处理的每一步
   - 确保可重现性
